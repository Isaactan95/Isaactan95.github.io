After coming to the office as a DEV, what's next?

lets start by cloning the project.

But how...?

1. Clonning a project from GitHub or GitLab


							$ git clone [url]

git clone [url]

Example: 


							$ git clone https://github.com/company/project.git
							cloning into 'project'...
							remote: Counting objects: 45, done.
							remote: Compressingobjects: 100% (120/120), done.
							remote: Total 45 (delta 5), reused 45 (delta 5), pack-reused 0
							unpacking objects: 100% (120/120), done.

And so you begin to code your beautiful ideas...

After you edited a file or added new file, what's next?

You need to push the code to the code base.

But how...?

2. Adding files to stagging area


							$ git add [files]

git add [file]

This command will add the files that was passed in as argument to the staging area.


							$ git add somefile.cs

git add * / -all / -a

This command will add all the changed files to the staging area.


							git add *

After you staged your changes files...

It's time...

To commit...your life...

I mean your files...

3. Commit the changes with a message.


							git commit -m "[ Type in the commit message ]"

git commit -m "[ commit message ]"

Example


							$ git commit -m  "[Project][210098] added xyz features."
							[master (root-commit) af23869] [Project][210098] added xyz features.
							3 files changed, 10,549 insertions(+), 5933 deletions(-)
							create mode 100644 project/index.html
							create mode 100644 project/css/index.css
							create mode 100644 project/js/index.js

Commit message standard:


							"[project][43212345] added xyz features."

- The [project] is the project name that was assigned to you

- The [43212345] is the ticket number for the project, it may vary due to different ticket's number.

MUST FOLLOW!!!!

After you commited your life...

I mean your files...

Time to...?

Yes...its time to push.

4. Push the changes to the code base.


							$ git push

git push

Yay, you did it!!!

You pushed your code to the code base. Now inform your team leader on your code push.

Congratulations on your first Git actions.

You have completed the basic of Git.

Hold up...

If you are to edit some code that has been pushed recently, how would you get the changes files?

It is not automatically downloaded or synced with your local repos.

5. Pulling changes to local repository.


							$ git pull

git pull


							$ git pull
							remote: counting objects: 13 done.
							remote: compressing objects: 100% (8/8), done.
							remote: Total 13 (delta 1) , reused 10 (delta 1) pack-reused 0
							Unpacking objects: 100%(13/13) done.
							From https://github.com/username/project
							* branch 		HEAD		->FETCH_HEAD

The normal flow of daily coding is:


						$ git pull
						// after you write your code
						$ git add * 
						$ git commit -m "message"
						$ git push
